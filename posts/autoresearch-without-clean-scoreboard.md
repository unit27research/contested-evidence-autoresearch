# Autoresearch Without A Clean Scoreboard

Autoresearch works best when the loop can be scored.

Change the code. Run the experiment. Check the benchmark. Keep the improvement. Repeat.

That shape is powerful when the feedback signal is real. It is much weaker when the disputed thing is not the score, but what the score means.

Some fields do not have a clean scoreboard. The evidence is mixed. Sources disagree. The same words point to different constructs. A strong result may hold only for one population, time period, method, or scope condition.

In those settings, the loop should not optimize toward the most confident answer. It should first improve the evidence environment:

- separate findings from interpretations
- bind claims to sources
- preserve credible conflicts
- make scope conditions explicit
- assign someone to argue against the leading theory
- generate next questions from unresolved disagreement

That is the idea behind Contested Evidence Autoresearch.

It is not an autonomous scientist. It is not a systematic review replacement. It is not a truth machine.

It is a Markdown field kit for domains where the useful output is structured uncertainty with a ranked attack plan.

The central question is simple:

> What should a research loop optimize when there is no clean validation curve?

My current answer: disagreement quality, source traceability, theory vulnerability, and next-question value.
